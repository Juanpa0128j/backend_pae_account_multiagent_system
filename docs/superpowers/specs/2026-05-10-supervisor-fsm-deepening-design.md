# Supervisor FSM Deepening — Batch 3 Spec

## Date: 2026-05-10

## Goal
Extract the Supervisor FSM (`app/agents/supervisor.py`, 1,055 lines) into a thin dispatcher plus per-pipeline router modules, with zero behavioral regressions.

## Problem Statement
The supervisor node currently mixes routing decisions with file parsing, LLM classification, DB session management, and complex retry logic. This makes it impossible to test routing without mocking LlamaParse, Gemini, and Postgres.

## Design

### Module Structure

```
app/agents/routing/
├── __init__.py              # Backward-compat re-exports
├── supervisor_node.py       # Thin dispatcher (~100 lines)
├── ingest_router.py         # Ingest pipeline routing
├── process_router.py        # Process pipeline routing + retry loops
├── reporting_router.py      # Trivial reportero dispatch
├── terminal_nodes.py        # error_terminal, review_terminal, audit_review_terminal
└── edge_functions.py        # should_retry_agent, should_retry_contador, should_retry_auditor

app/services/
├── file_preview.py          # extract_preview(file_path) → text_preview, parsed_content
├── classification_persistence.py  # save_classification(ingest_id, classification)
└── tributario_normalizer.py # normalize_tributario_output(state, output) → dict
```

### Backward-Compatibility

`app/agents/supervisor.py` becomes a thin re-export module:

```python
from app.agents.routing import (
    supervisor_node,
    route_after_supervisor,
    should_retry_agent,
    should_retry_contador,
    should_retry_auditor,
    error_terminal_node,
    review_terminal_node,
    audit_review_terminal_node,
)
from app.agents.validation_rules import (
    validate_output_node,
    validate_contador_output_node,
    validate_auditor_output_node,
)
```

No changes to:
- `app/agents/graph.py`
- `app/api/v1/*`
- Any test files (except new tests in `tests/agents/routing/`)

### Zero-Regression Strategy

Extract in 3 vertical slices, each testable independently:

**Commit 1: Terminal nodes + edge functions**
- Move terminal nodes → `routing/terminal_nodes.py`
- Move edge functions → `routing/edge_functions.py`
- `supervisor.py` imports from new modules
- Tests: `tests/agents/routing/test_terminal_nodes.py`, `test_edge_functions.py`

**Commit 2: Ingest router extraction**
- Move file validation, preview extraction, classification, DB persistence, pathway resolution → `routing/ingest_router.py`
- Extract `file_preview.extract_preview()` adapter (wraps LlamaParse, openpyxl, xml)
- `supervisor_node` delegates ingest mode to `ingest_router.route(state)`
- Tests: `tests/agents/routing/test_ingest_router.py`, `tests/services/test_file_preview.py`

**Commit 3: Process router extraction**
- Move contador/tributario/auditor routing + retry loops → `routing/process_router.py`
- Extract `tributario_normalizer.normalize()` as pure function
- `supervisor_node` delegates process mode to `process_router.route(state)`
- Tests: `tests/agents/routing/test_process_router.py`, `tests/services/test_tributario_normalizer.py`

### Safety Rules

- No behavioral changes: same DB session patterns, same retry budgets, same caps, same logging, same state mutations
- No changes to `graph.py`, API layer, or existing tests
- Each commit must pass `make test` and `make lint` before proceeding

## Testing Strategy

- Unit tests for each extracted router module using plain dicts and mocks
- Integration tests verify the thin dispatcher still routes correctly
- Existing test suite must pass unchanged after each commit
